# Для преподавателя (спойлеры)

Не класть участникам вместе с условием.

## Timing Gate (`timing_gate.exe`)

Цифры: 2, 7, 1, 8 по порядку после старта сессии.

Между нажатиями проверяются задержки в мс. Если цифры верные, а паузы нет — будет «почти верно» и левый флаг.

Победа: попап с нормальным `flag{...}`.

## ImGui Vault (`vault_fresh.exe`)

Есть подставной PIN из подсказки и настоящий для текущего профиля. Профили меняются в зависимости от `vault.dat` и того, ушёл ли человек в блокировку и перезапуск.

Бледные надписи на прогресс-баре — часть подсказки.

После правильного PIN флаг в hex, XOR ключ в интерфейсе в нужной ветке.

## VOID CONFIG (`eclipse_config.exe`)

В интерфейсе есть как минимум три исхода:
- отказ (неверная конфигурация),
- ложный «успех» (декой-ветка),
- настоящий успех (реальный флаг).

Проверяется кортеж параметров (Enable/FOV/Mode) через хэш-проверку. Отдельные параметры могут быть отвлекающими и не участвовать в валидной ветке.

Для зачёта принимать только флаг из настоящего окна успеха (см. `FLAGS.md`).

Код 3-го задания:
- `Task-03-VOID-CONFIG/` (основной вариант)
- `Source/EclipseConfig/` (упрощённый вариант)

Полный райтап и все флаги: [WRITEUPS.md](WRITEUPS.md), таблица флагов: [FLAGS.md](FLAGS.md).
