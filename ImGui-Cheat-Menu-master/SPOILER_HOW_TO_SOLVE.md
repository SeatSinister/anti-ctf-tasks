# ⚠️ СПОЙЛЕР: как решать VOID CONFIG (для организаторов)

## Ловушка «ложный комментарий»

В `main.cpp` есть комментарий с «эталоном сервера»: модуль **OFF**, FOV **180**, режим **Выкл.**, **noise_floor = 127**.

Если выставить ровно это и нажать «ПРИМЕНИТЬ», откроется окно **«УСПЕХ»**, но это **канал-ловушка**: показывается строка из `kFake` (XOR 0x5A), **не принимается** как настоящий флаг.

## Настоящая проверка

1. Константа эталона **разбита XOR** на три части:
   - `void_seal.h` → `vs::kTelemetrySaltA`
   - `void_seal_frag.h` → `vsf::kTelemetrySaltB`, `vsf::kTelemetrySaltC`
   - Сборка: `kTelemetrySaltA ^ kTelemetrySaltB ^ kTelemetrySaltC == 0xA34AF444`

2. Функция `vs::TmPacketHash(on, fov, mode)` — это **FNV-1a** по байтам:
   - `1` если модуль включён, иначе `0`
   - младший байт FOV, старший байт FOV
   - индекс режима (0…4)

3. Условие успеха: `TmPacketHash(...) == 0xA34AF444`.

## Подбор кортежа (без ручного перебора в UI)

```python
def tm_hash(on, fov, mode):
    h = 0x811C9DC5
    def feed(b):
        nonlocal h
        h = ((h ^ b) * 0x01000193) & 0xFFFFFFFF
    feed(1 if on else 0)
    feed(fov & 0xFF)
    feed((fov >> 8) & 0xFF)
    feed(mode & 0xFF)
    return h

TARGET = 0xA34AF444
for on in (0, 1):
    for fov in range(361):
        for mode in range(5):
            if tm_hash(bool(on), fov, mode) == TARGET:
                print(on, fov, mode)
```

Результат: **модуль включён**, **FOV = 271**, **режим = 4** («Полный»).  
`noise_floor` на результат **не влияет**.

## В интерфейсе

- Включить модуль (чекбокс +/или большая кнопка).
- FOV **271** (удобно: Ctrl+ЛКМ по ползунку для точного ввода).
- Режим **«Полный»**.
- Нажать **«ПРИМЕНИТЬ КОНФИГ»** → окно успеха с **настоящим** флагом из `kReal` (XOR 0x5A).

## Что усложнено для ИИ / статического чтения

- Нет явного `271` и индекса `4` в условии `if`.
- Эталон не одной константой в `main.cpp`, а XOR трёх кусков в **двух заголовках**.
- Второй «успех» с **другой** строкой отвлекает от правильного состояния.
